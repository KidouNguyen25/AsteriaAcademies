Before doing any task in this repository:

1. Use projectmem MCP tools first.
2. Read instructions, summary, project map, active issues, recent decisions, and file warnings.
3. Before editing a file, check whether projectmem has warnings or failed attempts related to that file.
4. During work, log:
   - issues when found
   - failed attempts immediately after they fail
   - partial attempts when incomplete
   - confirmed fixes after verification
   - architectural decisions
   - fragile files or setup gotchas
5. At the end, update memory with:
   - what changed
   - what failed
   - what worked
   - files touched
   - next recommended step
6. Do not read or mix memory from other repositories unless explicitly instructed.
